# Security policy

## Supported versions

AxiomaticGW is a research-stage formalization. Security and correctness fixes
are applied to the `main` branch; older snapshots are not maintained as
separate supported release lines.

## Reporting a vulnerability

Do not report credentials, dependency vulnerabilities, or other sensitive
security issues in a public issue. Use GitHub's private vulnerability reporting
for the repository instead. Include the affected revision, impact, and a
minimal reproduction when possible.

Mathematical errors and ordinary Lean build failures are not security issues
and should be reported through the public issue tracker.
